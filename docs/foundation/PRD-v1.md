# PRD-v1 - aikai

## 1. 项目目标

构建可扩展的 AI 助手底座，并通过分阶段方式交付。
近期目标是在最小功能范围内快速上线，同时保留向企业能力升级的清晰路径。

## 2. 版本策略

- `V1.0`：单轮问答（单用户、个人版）
- `V1.1`：稳定性增强（错误处理、超时、可观测）
- `V1.2`：工程化加固（测试、环境分层、CI 基线）
- `V1.3`：多轮问答（会话与消息上下文）
- `V1.4`：记忆能力与租户就绪存储模型
- `V1.5`：平台边界冻结，为大版本升级做准备
- `V2.0`：知识库（RAG）+ MCP + 外部 API 集成

## 3. V1.0 范围（包含）

- 单轮问答请求与响应
- 单用户默认模式（`personal`）
- 极简前端聊天页面
- 极简后端问答接口
- 健康检查接口
- 基础结构化日志与错误处理

## 4. V1.0 范围（不包含）

- 多轮上下文
- 记忆管理
- 知识库文档接入与检索
- MCP 工具调用
- 完整组织管理

## 5. 核心约束（必须长期保持）

- 从第一版开始在核心数据模型中预留 `tenant_id`
- 系统配置预留 `edition`（`personal | enterprise`）
- 明确并保持模块边界：
  - `chat`
  - `memory`
  - `knowledge`
  - `tooling`
- 早期版本仅启用 `chat`，其余模块保留为扩展点

## 6. V1.0 成功标准

- 端到端问答链路稳定可用
- 前后端本地启动流程清晰
- 常见失败场景返回明确错误信息
- 架构可平滑升级到 `V1.3+` 能力
