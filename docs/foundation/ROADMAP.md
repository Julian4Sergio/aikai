# 版本路线与验收标准

## V1.0 - 单轮问答基线

### 范围

- 单轮提问与回答
- 单用户个人版模式
- 基础页面与基础接口

### 验收

- 用户可完成一次提问并收到回答
- 前后端可本地运行
- 每次请求有基础日志记录

## V1.1 - 稳定性与可运维性

### 范围

- 超时与重试策略基线
- 错误码体系优化
- 初始运行指标接入

### 验收

- 超时行为可预测且一致
- 主要异常场景返回标准化错误
- 核心接口延迟与错误率可观测

## V1.2 - 工程化基线

### 范围

- 单元测试与集成测试基线
- 多环境配置
- CI 中加入检查与测试

### 验收

- 关键路径测试在 CI 中稳定通过
- 至少支持 `dev` 与 `staging` 配置
- 代码质量检查实现自动化

## V1.3 - 多轮问答

### 范围

- 会话与消息模型
- 最近 N 轮上下文窗口

### 验收

- 用户可持续进行多轮对话
- 上下文可正确注入模型提示词

## V1.4 - 记忆与租户就绪基础

### 范围

- 记忆抽象层（短期/长期契约）
- 租户就绪数据结构与访问边界
- 个人版升级企业版的入口钩子

### 验收

- 数据模型具备租户隔离字段
- `edition` 配置可控制个人/企业能力入口

## V1.5 - 平台边界冻结

### 范围

- 模块接口稳定化
- 知识库与工具能力扩展点定稿

### 验收

- 内部契约文档完备且稳定
- `V2.0` 可在不重写核心的前提下推进

## V2.0 - 大版本能力升级

### 范围

- 知识库接入与检索（RAG）
- MCP 工具注册与调用
- 外部 API 集成与治理

### 验收

- 基于知识库的回答可追溯来源
- 工具调用基于 Schema，具备鉴权与超时策略
- 外部 API 通过标准适配层接入
