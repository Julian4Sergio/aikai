# V1.1 承接清单（来自 V1.0）

## 目的

本文件用于记录已实现或已预研、但超出 `V1.0` 严格范围的能力，并明确延期到 `V1.1` 处理。

## 从 V1.0 延期的事项

- 流式响应体验（基于 SSE 的增量渲染）
- 多轮会话上下文（`history` 注入）
- 生成停止/取消的交互优化
- 更完整的会话连续性前端交互

## V1.1 目标

- 保持 `V1.0` API 契约作为生产基线稳定可用
- 在 `V1.1` 以版本化契约和测试重新引入延期能力
- 在 `V1.x` 内保证向后兼容地发布

## 说明

- `V1.0` 仍是单轮问答、JSON 响应的基线版本。
- 延期项不应阻塞 `V1.0` 发布检查清单。
