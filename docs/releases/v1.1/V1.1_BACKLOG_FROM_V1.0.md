# V1.1 承接清单（来自 V1.0）

## 目的

本文件用于记录已实现或已预研、但超出 `V1.0` 严格范围的能力，并明确延期到 `V1.1` 处理。

## 从 V1.0 延期的事项

- 流式响应体验（基于 SSE 的增量渲染）
- 多轮会话上下文（`history` 注入）
- 生成停止/取消的交互优化
- 更完整的会话连续性前端交互

## V1.1 目标

- 保持 `V1.0` API 契约作为生产基线稳定可用
- 在 `V1.1` 以版本化契约和测试重新引入延期能力
- 在 `V1.x` 内保证向后兼容地发布
- 以用户体验提升为主 KPI，优先优化流式首包体验与交互连贯性

## V1.1 决策冻结（2026-02-14）

- 版本拆分：`V1.1.0` + `V1.1.1`
- 多轮策略：先实现“短窗口上下文”，不在 `V1.1` 引入复杂长期记忆
- 优先级：体验优先，`SSE` 流式传输进入 `V1.1.0`

## 里程碑范围

### V1.1.0（体验底座）

- `SSE` 流式输出（后端事件流 + 前端增量渲染）
- 生成中止/取消链路（前端可取消、后端可中止）
- 首包体验优化（关注首 token 时间）
- 向后兼容：保留 `V1.0` JSON 同步模式作为兼容路径

### V1.1.1（会话增强）

- 短窗口 `history` 注入（最近 N 轮）
- 会话连续性前端交互（续聊、刷新恢复的最小能力）
- 流式与会话能力联调回归

## 优先级（P0/P1/P2）

- `P0`: `SSE` 流式、取消链路、首包体验指标、契约兼容
- `P1`: 短窗口上下文注入、会话连续性前端交互
- `P2`: 长期记忆、复杂会话编排（延期到后续版本）

## 说明

- `V1.0` 仍是单轮问答、JSON 响应的基线版本。
- 延期项不应阻塞 `V1.0` 发布检查清单。
- 执行细节见 `docs/releases/v1.1/V1.1_EXECUTION_PLAN.md`。
